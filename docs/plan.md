**文档版本**: 1.2

**适用范围**: EZWowX2 项目组

## 1. 概述

### 1.1 计划目的

本文档提供 EZWowX2 项目重建的详细实施计划，包括任务分解、优先级排序、时间估算、依赖关系、里程碑节点、资源需求和风险缓解策略。

### 1.2 背景

EZWowX2 是一个魔兽世界 12.0 版本的非侵入性自动化辅助工具项目，采用多语言混合架构（Lua/C#/Python）。当前代码存在硬编码、模块耦合、状态管理混乱等问题，需要系统性重构。

### 1.3 重构目标

| 目标 | 描述 |
|-----|------|
| 业务连续性 | 零停机迁移，功能等价性 100% |
| 架构解耦 | 核心接口 100% 抽象，依赖注入 100% |
| 代码质量 | 代码重复率 ≤5%，魔法数字数量 = 0 |
| 可维护性 | 单元测试覆盖率 ≥60%，文档覆盖率 100% |

## 3. 详细任务分解

#### 阶段目标
建立核心接口抽象层，实现依赖注入容器，将硬编码配置外部化。

- 定义核心接口：
  - `ICapture` - 截图捕获接口
  - `IBridgeClient` - 数据桥接客户端接口
  - `IInputSender` - 输入发送接口
  - `INodeExtractor` - 节点提取器接口

  - `PixelFrame` - 像素帧（支持crop裁剪）

  - `ValidationResult` - 验证结果

**T1.6 完成（2026-04-03）**：

**T1.7 完成（2026-04-03）**：
- 创建 `EZDriverX2/EZDriverX2/config/node_layout.py`
- 实现 `RegionSlice`、`NodeRegion`、`NodeLayout` 不可变配置类
- 提供 `DEFAULT_NODE_LAYOUT` 默认布局

**T1.8 完成（2026-04-03）**：
- 创建 `EZDriverX2/EZDriverX2/config/occlusion_rules.py`
- 实现 `OcclusionRule`、`OcclusionCheckResult` 不可变数据类
- 实现 `OcclusionRuleEngine` 遮挡检测引擎
- 提供 `DEFAULT_OCCLUSION_RULES` 默认规则和 `create_default_engine()` 工厂函数

**T1.9 完成（2026-04-03）**：
- 创建 `EZDriverX2/EZDriverX2/config/validation.py`
- 实现 `validate_unit_data()`、`validate_spell_data()`、`validate_context_data()` 验证函数

**T1.2-T1.5 完成（2026-04-03）**：

  - `MockCapture` - ICapture Mock实现
