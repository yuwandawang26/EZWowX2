## 1. 文档概述

### 1.1 设计目标
为EZWowX2简易方案设计一个游戏外辅助浮窗界面，提供实时状态监控、按键日志和便捷控制功能。

### 1.2 项目背景与约束

**简易方案技术架构**：
```
┌──────────────────┐     ┌──────────────────────┐     ┌─────────────────────┐
│ C_AssistedCombat │────▶│ EZAssistedX2 (插件)   │────▶│ EZAssistedX2.PY/NET │
│   (官方API)       │     │ 将API结果转化为色块    │     │ 读取像素块，发送按键 │
└──────────────────┘     └──────────────────────┘     └─────────────────────┘
```

**核心机制**：
- 游戏内插件将战斗状态转化为**20种固定颜色块**
- 游戏外程序读取颜色 → 映射到组合键 → 发送按键到游戏

**关键约束**：
- KeyMapping只占用**组合键**（SHIFT/ALT + NUMPAD0-9）
- **所有单个按键均未被占用**，可自由配置为UI快捷键
- 当前EZAssistedX2.PY/.NET没有独立的浮窗UI

### 1.3 设计原则
- **简洁高效**: 只展示核心监控信息
- **非侵入性**: 半透明设计，不遮挡游戏画面
- **可靠性**: 状态信息清晰，异常可快速识别
- **易用性**: 一键控制，用户可自定义快捷键

## 5. 交互设计

### 5.1 一键启停功能

**核心特性**：
- 用户可自定义**单个字母键**作为启停快捷键（必须支持）
- 默认快捷键：**Q**
- 支持按键：所有单个字母键（Q,W,E,R,T,A,S,D,F,G,Z,X,C,V,B,N,M等）

**技术实现说明**：

> ⚠️ **重要更新 (v7.2)**: 由于keyboard库在某些环境下存在兼容性问题（特别是字母键如Q），已改用**混合方案**：
> 1. **首选**: Win32 Low-Level Keyboard Hook (WH_KEYBOARD_LL, 线程级别)
> 2. **备用**: GetAsyncKeyState 轮询模式 (30ms间隔, ~33Hz)
> 3. **跨线程通信**: 使用Qt Signal/Slot机制替代QTimer.singleShot
>
> 当前实际运行在**轮询模式**，响应延迟约30-60ms，满足使用需求。

### 5.2 快捷键修改流程

**快捷键配置规则**：
> 启停快捷键必须是**单个字母键**（如Q、W等），这是项目核心需求。
> 快捷键编辑器仅接受单个字母键作为输入。

```
1. 用户点击 [点击修改] 按钮
         ↓
2. 进入按键捕获模式，显示提示"请按下新的启停快捷键"
         ↓
3. 用户按下单个字母键（如 W）
         ↓
4. 系统验证：
   - 是否为单个字母键（A-Z）
   - 是否与KeyMapping冲突（不冲突，单个键未被占用）
         ↓
5. 有效 → 保存新快捷键，更新显示"当前: W"
   无效 → 显示错误提示，留在捕获模式
         ↓
6. 用户可点击 [取消] 退出
```

**捕获规则**：
- ✅ 仅接受**单个字母键**（A-Z, a-z）
- ❌ 不接受组合键（Ctrl+Q等）
- ❌ 不接受功能键（F1-F12）、数字键、NUMPAD
- ❌ 不接受已被KeyMapping占用的组合键

#### 5.3.1 启动流程
```
程序启动
    ↓
```

### 6.4 全局快捷键注册

**⚠️ 核心要求：必须支持单个字母键**

> 项目要求启停快捷键必须是**单个字母键**（如Q、W等）。
> Win32 RegisterHotKey无法满足此需求，必须使用keyboard钩子方案。

**最终技术方案：混合Win32 API + 轮询模式 (Python)**

| 方案 | 单个字母键 | 组合键支持 | 复杂度 | 稳定性 | 当前状态 |
|-----|-----------|-----------|--------|--------|---------|
| RegisterHotKey | ❌ 不支持 | ✅ 支持 | 简单 | 高 | 不采用 |
| keyboard库 (Python) | ⚠️ 部分支持 | ✅ 支持 | 简单 | 低 | ❌ 已废弃 |
| Low-Level Hook (C#) | ✅ 支持 | ✅ 支持 | 中等 | 高 | .NET版本备选 |
| **混合方案 (Hook+轮询)** | ✅ 支持 | ✅ 支持 | 中等 | **高** | **✅ 当前使用** |

**当前实现（hotkey_manager.py）**：

```python
class GlobalHotkeyManager:
    """
    全局快捷键管理器 - 混合方案
    1. 首选: Win32 Low-Level Keyboard Hook (线程级别)
    2. 备用: GetAsyncKeyState 轮询检测 (30ms间隔)
    """

    def start_listening(self, blocking: bool = True):
        hook_installed = self._try_install_hook()
        if hook_installed:
            # Hook模式
            self._run_message_loop()
        else:
            # 轮询模式 (30ms, 300ms防抖)
            self._run_polling_loop()

    def _run_polling_loop(self):
        """轮询模式: 使用GetAsyncKeyState定期检查按键状态"""
        poll_interval = 0.03  # 30ms (~33Hz)
        debounce_interval = 0.3  # 300ms防抖

        while self._running:
            for key_name in self._single_key_callbacks:
                state = user32.GetAsyncKeyState(vk_code)
                is_pressed = (state & 0x8000) != 0
                # ... 触发回调
```

## 7. 用户场景

### 7.1 场景一：快速启停

1. 用户启动程序，浮窗显示在左上角
2. 选择游戏窗口，按下 **Q** 开始
3. 浮窗实时显示检测到的按键日志
4. 再次按下 **Q** 暂停
5. 无需切换窗口或点击按钮

1. 用户按下 **Ctrl+Shift+E** 展开设置面板

3. 按下新字母键（如 W）
4. 系统验证并提示"快捷键已更新为 W"
5. 现在 W 成为新的启停快捷键
6. 设置自动保存

### 7.3 场景三：临时隐藏

1. 用户按下 **Ctrl+Shift+H** 隐藏浮窗
2. 浮窗消失，但程序继续运行
3. 按下 **Ctrl+Shift+H** 再次显示
4. 或右键托盘图标选择"显示"

**核心要求：必须支持单个字母键作为启停快捷键**

> 项目核心需求，启停功能必须绑定到**单个字母键**（如Q、W等）。
> Win32 RegisterHotKey无法满足，必须使用keyboard库或Low-Level Hook。

**最终确定的UI快捷键**：

| 功能 | 快捷键 | 实现方式 | 状态 |
|-----|--------|---------|------|
| 启停 | **Q** (单个字母键) | keyboard库 / Low-Level Hook | ✅ 必选 |
| 显示/隐藏 | Ctrl+Shift+H | RegisterHotKey | 可选 |
| 展开/收起 | Ctrl+Shift+E | RegisterHotKey | 可选 |

**支持的配置范围**：
- 单个字母键：Q, W, E, R, T, A, S, D, F, G, Z, X, C, V, B, N, M
- 功能键：Tab, 空格

**不可用**：
| 按键类型 | 原因 |
|---------|------|
| NUMPAD0-9 | KeyMapping占用 |
| F5-F11 | KeyMapping占用 |

### 10.2 与原设计方案对比

|-----|----------|-------------|------|

### 12.3 技术实现路径

#### 12.3.1 技能名称管理器

```python
import json
from pathlib import Path

class SkillNameManager:
    def __init__(self, config_path: str = "skill_names.json"):
        self.config_path = Path(config_path)
        self.skills = self._load_config()

    def _load_config(self) -> dict:
        if self.config_path.exists():
            with open(self.config_path, 'r', encoding='utf-8') as f:
                return json.load(f)
        return self._default_config()

    def _default_config(self) -> dict:
        return {
            "version": 1,
            "skills": {k: {"name": v, "enabled": True} 
                      for k, v in DEFAULT_SKILL_NAMES.items()}
        }

    def get_name(self, color_key: str) -> str:
        return self.skills["skills"].get(color_key, {}).get("name", "未知")

    def set_name(self, color_key: str, name: str):
        if color_key in self.skills["skills"]:
            self.skills["skills"][color_key]["name"] = name
            self._save_config()

    def reset_all(self):
        self.skills = self._default_config()
        self._save_config()

    def _save_config(self):
        with open(self.config_path, 'w', encoding='utf-8') as f:
            json.dump(self.skills, f, ensure_ascii=False, indent=2)
```

```python
def validate_skill_name(name: str) -> tuple[bool, str]:
    """验证技能名称，返回(是否有效, 错误信息)"""
    if not name:
        return False, "名称不能为空"

    name = name.strip()

    if len(name) < 1:
        return False, "名称至少1个字符"
    if len(name) > 8:
        return False, "名称最多8个字符"

    import re
    if not re.match(r'^[\w\-\(\)\[\]]+$', name):
        return False, "只允许中文、英文、数字和-_()[]"

    return True, ""
```
