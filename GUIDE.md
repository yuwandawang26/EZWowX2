## 1. 项目概述

### 1.1 什么是 EZWowX2？

EZWowX2 是一个适用于《魔兽世界》12.0+版本的**非侵入性自动化辅助工具示例**。本项目采用**像素化技术**将游戏内数据转换为外部程序可读的格式，实现技能自动释放。

> ⚠️ **重要提示**: 本项目为开源示例，旨在展示技术思路。使用前请务必阅读[免责声明](#⚠️-免责声明)。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 🎮 **自动技能释放** | 检测游戏内颜色块，自动发送对应按键 |
| 🖥️ **悬浮窗UI** | 半透明浮窗，实时显示运行状态和按键日志 |
| ⌨️ **全局快捷键** | Q键启停、Ctrl+Shift+E/H控制窗口 |
| 🔐 **管理员权限** | 自动请求UAC提权，确保Win32 API正常工作 |
| 🎨 **自定义图标** | 程序图标、托盘图标统一设计 |
| 📝 **智能日志** | 默认关闭DEBUG模式，可通过📝按钮切换 |
| 🔄 **自动重试** | 首次启动时自动等待战斗开始（最长20秒） |

### 1.3 适用场景

- ✅ 适合插件开发入门学习
- ✅ 适合了解像素自动化技术原理
- ✅ 适合作为AI重构的基础项目

### 1.4 技术要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 1903+ / Windows 11 |
| Python | 3.12+ |
| 游戏版本 | 魔兽世界 12.0+ |

| 窗口模式 | 推荐"窗口化全屏"或"窗口模式" |

## 4. 使用指南

### 4.1 基本操作

#### 启动辅助

1. 确保魔兽世界已启动并登录
2. 在浮窗下拉框中选择正确的WoW窗口
3. 按 **Q键** 或点击播放按钮
4. 程序会自动等待战斗开始（最多20秒）
5. 右键攻击敌人进入战斗后自动释放技能

#### 暂停/恢复

- 按 **Q键** 切换暂停/恢复
- 或点击浮窗的 ⏸/▶ 按钮

#### 停止辅助

- 再次按 **Q键** 从暂停状态停止
- 或点击浮窗的 ✕ 关闭按钮

### 4.2 高级配置

#### 设置文件位置

| 配置文件 | 路径 | 说明 |
|---------|------|------|
| settings.json | `EZAssistedX2.PY/settings.json` | 程序设置 |

#### 手动调整设置

`settings.json` 示例:
```json
{
    "hotkey": "q",
    "window_position": null,
    "window_expanded": false,
    "selected_window": null
}
```

### 4.3 多显示器支持

程序通过窗口句柄定位游戏窗口，支持多显示器环境。只需在下拉框中选择正确的WoW窗口即可。

### 4.4 性能优化建议

| 建议 | 效果 |
|------|------|
| 关闭HDR | 避免像素读取异常 |
| 关闭插帧/FreeSync | 避免帧率不稳定 |
| 使用窗口化全屏 | 推荐模式，兼容性最好 |
| 关闭DEBUG日志 | 减少磁盘IO，降低CPU占用 |

## 5. 常见问题 (FAQ)

### Q1: 为什么首次启动没有技能输出？

**现象**: 按Q后显示"运行中"，但没有技能释放

**原因**: 程序需要检测到游戏内的颜色块区域才能开始工作。如果你不在战斗中，颜色块不会出现。

**解决方案**:
1. 按Q启动后，程序会自动等待（最多20秒）
2. 走到敌人附近，右键攻击进入战斗
3. 程序检测到颜色块后会自动开始释放技能
4. 无需再次按Q！

**技术细节**: 参见 [故障排除 #T1](#t1-首次启动无技能输出workerthread立即退出)

---

### Q2: 快捷键没有反应？

**可能原因及解决方案**:

| 原因 | 解决方案 |
|------|---------|
| 程序未以管理员权限运行 | 使用 Start.bat 启动（自动请求权限） |
| 其他程序占用快捷键 | 检查是否有其他软件使用相同快捷键 |
| 游戏在前台时某些键被拦截 | 尝试让浮窗在前台时按快捷键 |

**当前实现方式**: 
- 单键(Q): GetAsyncKeyState 轮询模式 (30ms间隔)
- 组合键(Ctrl+Shift): 同上轮询模式
- 预期延迟: 30-60ms

---

### Q3: CMD窗口一直显示/刷新？

**原因**: 使用了 `python.exe` 而非 `pythonw.exe`

**解决方案**: 
- 使用 `Start.bat` 启动（已优化为无CMD窗口）
- 或手动运行: `pythonw EZAssistedX2.py`

---

### Q4: 日志文件太大？

**现状对比**:

| 模式 | 4小时日志大小 | 说明 |
|------|-------------|------|
| OFF (默认) | <5MB | 仅记录WARNING及以上 |
| ON (调试) | 200-400MB | 记录所有DEBUG信息 |

**解决方案**:
1. 默认就是关闭的，无需担心
2. 如需排查问题，点击📝按钮临时开启
3. 问题解决后记得关闭

---

### Q5: 如何更换程序图标？

**方法一: 使用自带工具**
```bash
python generate_icon.py
```
然后编辑 `generate_icon.py` 中的颜色参数重新生成。

**方法二: 手动替换**
1. 准备一个256×256的PNG图片
2. 转换为ICO格式（可用在线工具）
3. 替换 `EZAssistedX2.ico`
4. 重启程序

---

### Q6: 支持哪些职业/专精？

**当前版本**: 通用型辅助工具，通过颜色块识别技能（HSL色轮渐变色系，21个槽位）

**支持的按键映射组**:
- SHIFT + NUMPAD (1~0): 绿色→品红渐变区（10个槽位）
- ALT + NUMPAD (1~0): 品红→黄绿渐变区（10个槽位）
- 纯白/纯黑: 特殊状态指示（不发送按键）

## 6. 故障排除

### T1: 首次启动无技能输出 (WorkerThread立即退出)

**错误日志**:
```
20:42:07 - WorkerThread已启动
20:42:08 - Worker已完成  ← 仅0.3秒后退出！
```

**根本原因**: 
WorkerThread在找不到颜色块模板时立即退出，而非等待战斗开始。

**修复状态**: ✅ 已在 v2026-04-03 修复

**修复内容**:
- 添加20次重试机制（每次间隔1秒）
- 自动等待战斗开始
- 显示详细的状态日志

**验证方法**:
1. 启动程序，不在战斗中按Q
2. 观察：应显示"运行中"并持续等待
3. 进入战斗后：自动开始释放技能

---

### T2: 找不到魔兽世界窗口

**错误信息**: "未找到游戏窗口"

**解决方案**:
1. 确认魔兽世界已启动并登录
2. 确认不是在加载界面/角色选择界面
3. 手动在下拉框刷新窗口列表
4. 如果是多开，确认选择了正确的窗口

---

### T3: 颜色块模板匹配失败

**错误信息**: "未找到模板" 或 "模板大小不正确"

**可能原因**:
1. 游戏分辨率/缩放比例改变
2. UI缩放设置变更
3. 插件未正确加载或配置

**解决方案**:
1. 检查游戏内插件是否启用
2. 确认UI缩放比例为100%
3. 重启游戏和辅助程序

---

### T4: 权限不足错误

**错误信息**: Win32 API调用失败

**解决方案**:
1. 必须以管理员权限运行
2. 使用 `Start.bat` 启动（自动请求权限）
3. 或右键 → 以管理员身份运行

---

### T5: 屏幕捕获失败

**错误信息**: "无法抓取屏幕帧"

**可能原因**:
1. DXGI版本不支持（需Windows 10 1903+）
2. 显卡驱动过旧
3. HDR/插帧功能开启

**解决方案**:
1. 更新Windows到最新版本
2. 更新显卡驱动
3. 关闭HDR和插帧功能
4. 重启程序

---

### T6: 托盘图标不显示

**检查项**:
1. `EZAssistedX2.ico` 文件是否存在
2. 图标格式是否正确（ICO格式）
3. 系统托盘是否被隐藏

**解决方法**:
```bash
python generate_icon.py  # 重新生成图标
```

### 7.4 关键技术选型

| 技术 | 选择 | 原因 |
|------|------|------|
| GUI框架 | PySide6 (Qt6) | 跨平台、丰富控件 |
| 屏幕捕获 | dxcam (DXGI) | 高性能、低延迟 |
| 图像处理 | OpenCV + NumPy | 成熟稳定 |
| 按键发送 | Win32 PostMessage | 直接发送到目标窗口 |
| 快捷键 | 混合Hook+轮询模式 | Hook优先，兼容性最佳 |
| 跨线程通信 | Qt Signal/Slot | 线程安全、自动队列 |
| 配置存储 | JSON | 人类可读、易于编辑 |
| 日志系统 | logging + SmartLogManager | 灵活可控 |

### 8.4 编码规范摘要

基于 PEP 8 的 Python 编码规范：

| 规范 | 要求 |
|------|------|
| 命名 | 模块/函数小写下划线，类名CapWords |
| 类型注解 | 公共API必须添加 |
| docstring | 公共类/函数必须有 |
| 异常处理 | 禁止裸露except，保留异常链 |
| 导入顺序 | 标准库 → 第三方 → 本项目 |
| 线程安全 | 共享状态必须加锁 |

详细规范见 [CODING_STANDARDS.md](docs/CODING_STANDARDS.md)

| 限制 | 影响 | 计划 |
|------|------|------|

| 不支持全屏独占 | 需要窗口模式 | 可接受 |

### C. 错误场景速查

> ⚠️ **注意**: 当前版本使用描述性错误消息字符串，未引入正式错误码常量体系。以下为错误场景与对应消息对照表。

| 场景 | 错误消息示例 | 触发位置 | 对应故障排除 |
|------|-------------|---------|------------|
| 屏幕捕获失败 | "无法抓取屏幕帧" | WorkerThread.run() | 检查DXGI兼容性 / [T5](#t5-屏幕捕获失败) |
| 未找到模板 | "初始化失败: N次尝试后仍未检测到颜色块区域" | WorkerThread.run() | 检查插件/UI设置 / [T3](#t3-颜色块模板匹配失败) |
| 模板大小不正确 | "模板大小不正确: NxM (期望: 12x4)" | WorkerThread.run() | 检查分辨率/缩放 / [T3](#t3-颜色块模板匹配失败) |
| 初始化超时 | "初始化失败: 20次尝试后..." | WorkerThread.run() | 确认已在战斗中 / [T1](#t1-首次启动无技能输出workerthread立即退出) |
| 窗口无效 | "窗口无效" 或 "目标窗口无效或已关闭" | EZWowX2App._on_pause_click() | 重新选择窗口 / [T2](#t2-找不到魔兽世界窗口) |
| 未找到游戏 | "未找到游戏" | EZWowX2App._on_pause_click() | 确认WoW已启动 / [T2](#t2-找不到魔兽世界窗口) |
| 权限不足 | UAC提示框（程序启动时） | main() → is_admin() | 以管理员身份运行 / [T4](#t4-权限不足错误) |
